Team > Lock

Using the new features in Subversion 1.2, you can lock one or more files for editing. This is useful for ensuring that two team members do not waste each others' time by editing the same resource if it is hard or impossible to merge these changes (for instance with binary files).

The lock command is issued from the 'Team' context menu of the resource.

Once locked by you, the resource has a check-mark next to it.

If you commit a change to a locked file, the lock is usuall unlocked, unless you select 'Keep locks' in the commit dialog. Alternatively, you can unlock the resource again.

Subversion also makes it possible to steal a lock, if a different user has locked a file and you need to take it over. This is accomplished by selecting the 'Steal Lock' option found in the dialog.

TIP: If you want to ensure that a specific file is always locked before committing, you can set the Subversion property svn:needs-lock on the file (and commit). This ensures that a lock must be taken before committing it.

Tasks:

Typical work cycle: Edit, Update, Commit
