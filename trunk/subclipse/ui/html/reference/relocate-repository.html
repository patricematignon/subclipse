Relocate Repository

Overview

Sometimes an administrator might change the base location of your repository — in other words, the contents of the repository doesn't change, but the URL used to reach the root of the repository does. For example, the hostname may change, or the URL schema, or perhaps just the path which leads to the repository. Rather than checkout a new working copy, you can use the Relocate option to have SVN rewrite the the beginnings of all the URLs in your working copy. No file contents are changed, nor is the repository contacted.

Process

Type the new repository URL.

The Relocate command should only used if the URL of the repository root has changed. This is a very infrequently used operation and should not be used to move to another SVN repository, nor should it be used to switch to another branch or directory within the same repository. An improperly used Relocate command may corrupt your working copy! The chances are, if you are unsure whether Relocate is the correct operation, it probably is not.
