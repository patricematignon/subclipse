SvnClientAdapter is a high level java API for subversion.

SvnClientAdapter can use either

  • javahl interface (uses JNI) : the faster approach

  • command line interface



Why should I use SvnClientAdapter instead of javahl ?

SvnClientAdapter is a higher level API (which uses javahl ...). That's why svnClientAdapter is easier to use in some cases.
For example, you can use ISVNClientAdapter addToIgnoredPatterns method to add a pattern of files to ignore to a directory.

public abstract void addToIgnoredPatterns(File path, String pattern)

throws SVNClientException;

With javahl, it would have been necessary to get the svn:ignore property, parse it, verify that the pattern is not already set, add the pattern to the property string and then set the property.


Same remarks apply for setting keywords substitution.
