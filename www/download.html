Downloads and Installation Instructions

See the changelog for the latest version of Subclipse. Unless otherwise noted, the best available version is always the most recently released version.

Detailed installation instructions for Subclipse in Eclipse 3.x can be found on the page Installation Instructions. Generally speaking, Subclipse is installed and updated from within Eclipse. There is no specific external download needed. However a zipped version of the Subclipse update site is always available for download here. Likewise, a zipped source distribution is available from the same location.

For all svnAnt, svnClientAdapter and Eclipse 2.x, downloads can be found here: http://subclipse.tigris.org/servlets/ProjectDocumentList

The Eclipse 2.x development line is no longer active. We are no longer fixing bugs in Eclipse 2.x nor making Eclipse 2.x releases. Only the discovery of a horrible data-loss or security bug would prompt us to do another Eclipse 2.x release. If you discover a bug in Eclipse 2.x, it is likely already fixed in a later development line. If you report the bug, you'll probably be told to upgrade first. Likewise, the same can all be said for Subversion 1.1.x and earlier. The last release of Subclipse that supported Subversion 1.1.x was the 0.9.30 release. Due to changes in the JavaHL and command-line interface in Subversion 1.2 it was not practical to continue to support both releases.
